# Shared Publish Flow

## Purpose

Unify the publish progress modal and delete confirmation patterns across all resource types (upstream, route, plugin_config, global_rule, static_resource, plugin_metadata).

## ADDED Requirements

### Requirement: Unified publish function
All resource composables SHALL use a shared `executePublish()` function from `useClusterUtils.ts` instead of duplicating the progress modal and edge result logic inline.

#### Scenario: publishResource calls executePublish
- **WHEN** any resource composable's publish function is called
- **THEN** it delegates to `executePublish` with params: `title`, `apiEndpoint`, `nodeIds`, `refreshFn`
- **THEN** `Modal.info` with `buildDeleteProgressContent` creates the progress modal
- **THEN** after API response, it parses `data.status` and shows success/partial/error result

### Requirement: Unified delete confirmation
All resource composables SHALL use `showDeleteConfirm` from `useClusterUtils.ts` for delete confirmation before executing the delete operation.

#### Scenario: deleteResource uses showDeleteConfirm
- **WHEN** any resource composable's delete function is called
- **THEN** it calls `showDeleteConfirm` with `title`, `apiEndpoint`, `nodes`, and an `onOk` callback
- **THEN** the callback processes DB/Edge deletion results through the progress modal

### Requirement: Remove dead code from ClusterList.vue
ClusterList.vue SHALL have all 7 inline modal/drawer components removed, along with their unused script destructuring.

#### Scenario: Inline modals removed
- **WHEN** ClusterList.vue is rendered
- **THEN** no duplicate route/pluginConfig/globalRule/staticResource inline modals or drawers exist
- **THEN** no duplicate VersionManagementModal exists
- **THEN** unused composable destructuring is removed

All resource composables SHALL use a shared `buildDeleteProgressHandler()` factory from `useClusterUtils.ts` to generate the `onOk` callback for `showDeleteConfirm`.

#### Scenario: deleteResource uses buildDeleteProgressHandler
- **WHEN** any resource composable's delete function calls `showDeleteConfirm`
- **THEN** `onOk` callback is generated by `buildDeleteProgressHandler` with params: `resourceName`, `apiEndpoint`, `cluster`, `refreshFn`, `clearSelectedFn`
- **THEN** the callback contains the progress modal and edge result processing
