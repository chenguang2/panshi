## Context

用户管理页面（`UserList.vue`）存在 4 处 UI/UX 问题，全部为前端展示层面的修复，不涉及后端 API 变更。

### 当前状态

- 表格列未设置宽度约束，角色列和状态列自适应导致列宽过大
- 权限列对非管理员用户始终显示"在编辑中设置"占位文本
- 权限列表中"边缘节点管理"命名与侧边栏菜单"Edge直连"不一致
- 重置密码输入框 `width: 100%` 导致在表单行中占据过多空间

### 设计稿参考

设计稿 `users.html` 展示了正确的样式标准：
- 角色用 `role-badge` 紧凑标签显示（管理员/普通用户）
- 状态用 `badge` 标签显示（启用/禁用）
- 权限用 `perm-tag` 小标签显示具体权限名
- 重置密码输入框与按钮在同一行紧凑排列

## Goals / Non-Goals

**Goals:**
- 角色列和状态列宽度与设计稿一致，紧凑展示
- 权限列显示用户实际权限标签，而非占位文本
- "边缘节点管理"统一改为"Edge直连"
- 重置密码输入框和按钮在同一行显示

**Non-Goals:**
- 不修改后端 API
- 不修改用户管理功能逻辑
- 不修改其他页面
- 不涉及数据库模型变更

## Decisions

### 1. 列宽方案

**决策**：为角色列设置 `width: 100`，状态列设置 `width: 80`。

角色内容为"管理员"/"普通用户"加标签样式，状态内容为"启用"/"禁用"加状态点，均不需要太宽。

### 2. 权限列渲染

**决策**：将 `permissionKeyToLabel` 映射用于权限列显示。非管理员用户在表格中遍历 `record.permissions` 数组，通过映射表转换为中文标签，用 `perm-tag` 样式渲染。

如果用户没有权限（空数组），显示 `—` 占位。

### 3. 命名统一

**决策**：`allPermissions` 中 `edge_nodes` 的 label 从"边缘节点管理"改为"Edge直连"，`permissionKeyToLabel` 映射同步修改。

### 4. 重置密码输入框宽度

**决策**：密码输入框添加 `style="max-width: 220px"` 或 CSS 类约束宽度，确保与"重置密码"按钮在同一行内显示。

## Risks / Trade-offs

- 权限列展示需要后端返回 `permissions` 字段 — 当前接口已返回，无影响
- 列宽修改后若将来角色/状态文案变长可能需要调整 — 低风险，修改简单
