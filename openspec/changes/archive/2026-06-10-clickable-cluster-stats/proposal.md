## Why

集群管理页面（集群卡片）底部的统计数据目前是纯文本，用户无法直接点击跳转到对应的资源页面，需要手动导航并重新选择集群过滤条件，操作路径长，效率低。

## What Changes

- 集群卡片底部的每项统计数字改为可点击的链接
- 点击后跳转到对应的资源列表页面（路由列表、上游列表、节点列表等）
- 跳转时通过 URL query 参数（`?cluster_id=X`）传递当前集群 ID
- 各资源列表页面在接收到 `cluster_id` 参数后自动设置集群过滤下拉框，无需用户手动选择
- 涉及的统计项：节点（含健康/总数）、路由、上游、插件组、全局规则、插件元数据、静态资源
- 每项统计数字整体作为可点击链接，含节点统计的"健康/总数"格式

## Capabilities

### New Capabilities
- `cluster-stat-links`: 集群卡片统计数字改为可点击链接，跳转到对应资源页面并自动选中当前集群

### Modified Capabilities
- `cluster-card-grid`: 集群卡片的统计数据显示方式从纯文本改为可点击链接

## Impact

- 前端：`views/ClusterList.vue` — 卡片统计区域改为链接
- 前端：各资源列表页面 — 接收 `cluster_id` URL query 参数并自动设置过滤条件
- 涉及的路由配置可能需要调整以支持 query 参数传递
