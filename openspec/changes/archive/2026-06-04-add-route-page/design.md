## Context

设计稿 `docs/ui/Live-Artifact-4/routes.html` 展示独立路由管理页面，包含：
- 页面标题 + 集群筛选下拉 + 新建按钮（去掉「发布全部」）
- HTTP 方法 chip 筛选（全部/GET/POST/PUT/DELETE/PATCH）
- 搜索栏 + 发布状态下拉（已发布/未发布）
- 表格（名称/描述、URI、方法、集群、优先级、版本、创建时间、操作）
- 操作下拉菜单：复制路由、编辑、发布、版本管理、删除（无禁用）
- 新建/编辑 Modal（含所属集群字段）
- 分页

## Key Decisions

| 决策 | 选择 | 理由 |
|---|---|---|
| 框架 | 独立页面 + Modal 表单 | 与上游管理一致 |
| 表格 | 自定义 HTML table | 与上游管理一致，匹配设计稿 |
| 操作菜单 | 自定义 action-menu | 与上游管理一致 |
| 表单 | Ant Design `<a-modal>` + `<a-form>` | 与上游管理一致 |
| 数据复用 | `useClusterRoutes` composable | 复用现有逻辑 |
| 跨集群列表 | 新增 `GET /api/v1/routes` 后端接口 | 参考 upstreams 实现 |
| RouteFormModal 使用方式 | RouteList 使用，ClusterRoutes 保持内联 | 降低风险 |
| 筛选参数 | 支持 method（LIKE 匹配）和 publish_status（检查 ConfigVersion） | 按需求新增

## Differences from Design

- 设计稿有「发布全部」按钮 → 去掉
- 设计稿状态列为启用/禁用 → 改为「版本」列
- 设计稿全部状态下拉 → 改为「发布状态」（已发布/未发布）
- 设计稿有「禁用」操作菜单 → 去掉
- 设计稿没有「复制路由」操作菜单 → 增加
- 设计稿没有「版本管理」操作菜单 → 增加
