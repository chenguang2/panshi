## Context

SSL 证书卡片现有操作栏有查看/编辑/删除/发布/版本管理 5 个按钮。用户要下载文件必须先点查看进入弹窗再下载，步骤较多。

## Goals / Non-Goals

**Goals:**
- 卡片操作栏增加"下载"按钮
- 点击弹出文件选择对话框，可选 cert/key/sign_cert/sign_key
- 勾选后打包为 ZIP 下载
- 双证书显示 4 个选项，单证书显示 2 个选项

**Non-Goals:**
- 不替换现有查看界面的下载功能（两者并存）
- 不涉及后端变更（文件内容已在 API 响应中）

## Decisions

### 决策 1: ZIP 打包方式 — JSZip

使用 `jszip` 库实现 ZIP 打包。这是业界标准前端 ZIP 库，支持异步添加文件并生成 Blob。

```bash
npm install jszip
```

### 决策 2: 对话框 UI

```
┌─ 选择要下载的文件 ───────────────────┐
│                                       │
│  ☑ 加密证书 (cert.pem)                │
│  ☑ 加密私钥 (key.pem)                 │
│  ☐ 签名证书 (sign_cert.pem)  ← 双证书 │
│  ☐ 签名私钥 (sign_key.pem)   ← 双证书 │
│                                       │
│  文件名将以证书名称命名                 │
│                                       │
│          [取消]    [下载 ZIP]          │
└───────────────────────────────────────┘
```

### 决策 3: ZIP 文件名格式

`{证书名称}_certs.zip`，内部文件使用 `{证书名称}_{类型}.pem` 命名。

## Risks / Trade-offs

- 新增 `jszip` 依赖（约 400KB）
- 不需要后端变更
