# 路由插件管理讨论

## 背景

当前磐石 Admin 的插件管理存在以下问题：

1. **插件选择不直观**：目前使用下拉选择器（`<a-select>`）选择插件，用户无法直观了解可用插件列表
2. **插件编辑功能受限**：`PluginEditorDrawer.vue` 目前硬编码了 `proxy-rewrite` 插件的字段，无法通用化支持所有插件
3. **表单/JSON 双模式不统一**：不同组件各自实现了表单/JSON 切换，缺少统一的编辑体验

## 参考方案

### APISIX Dashboard 的做法

APISIX Dashboard 采用：
- 左侧插件列表（可搜索），右侧插件配置面板
- 点击插件后展开配置表单
- 支持直接编辑 JSON
- 按插件分类展示（限流、认证、转换等）

### 方案一：卡片式插件选择 + 通用 Schema 驱动表单

**核心思路**：参考 APISIX Dashboard，将下拉选择改为卡片网格展示，基于插件 Schema 自动生成配置表单。

**UI 布局**：
```
┌─────────────────────────────────────────────────────────────┐
│  插件管理                                      [表单] [JSON] │
├──────────────────────┬────────────────────────────────────┤
│  搜索: [________]    │  已选插件 (3)                        │
│                      │  ┌────────┐ ┌────────┐ ┌────────┐   │
│  ┌──────────────┐    │  │ ip-rest│ │  cors  │ │proxy-rw│   │
│  │ ip-restricti │    │  └────────┘ └────────┘ └────────┘   │
│  │  IP 访问限制  │    │                                     │
│  └──────────────┘    │  配置详情区                           │
│  ┌──────────────┐    │  ┌────────────────────────────────┐  │
│  │    cors      │    │  │ whitelist: [192.168.1.0/24]   │  │
│  │  跨域资源共享│    │  │                                │  │
│  └──────────────┘    │  │ [_disable: ○  ●]               │  │
│  ┌──────────────┐    │  └────────────────────────────────┘  │
│  │proxy-rewrite │    │                                     │
│  │   URI 重写   │    │                                     │
│  └──────────────┘    │                                     │
└──────────────────────┴────────────────────────────────────┘
```

**优点**：
- 插件一目了然，无需展开下拉
- 基于 Schema 自动生成表单，减少硬编码
- 支持拖拽排序

**缺点**：
- 实现复杂度较高
- 需要后端提供完整的插件 Schema

### 方案二：侧边抽屉式插件配置（推荐）

**核心思路**：保留现有卡片网格交互，修复插件编辑器通用性问题。

**UI 布局**：
```
┌─────────────────────────────────────────────────────────────┐
│  插件管理                                                   │
├─────────────────────────────────────────────────────────────┤
│  [+ 添加插件]  [搜索: ________]  [分类: 全部 ▾]              │
│                                                             │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │ ✓ ip-rest   │  │ ✓ cors      │  │ ✓ proxy-rw │  [+ 添加] │
│  │ 已配置      │  │ 已配置      │  │ 已配置      │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
│                                                             │
│  当点击某个插件卡片 → 打开通用配置 Drawer                     │
└─────────────────────────────────────────────────────────────┘

Drawer 内容：
┌─────────────────────────────────────────────────────────────┐
│  配置插件: proxy-rewrite                        [表单] [JSON] │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  URI 重写:                                                  │
│  ┌─────────────────────────────────────────────────────┐    │
│  │ /old-api/(.*)                                       │    │
│  └─────────────────────────────────────────────────────┘    │
│                                                             │
│  请求头设置:                                                 │
│  set   X-Api-Version → v2                    [删除]        │
│        [header 名称] → [header 值]           [添加]        │
│                                                             │
│  add   X-Request-ID → 12345                   [删除]        │
│                                                             │
│  remove X-Legacy-Token                          [删除]       │
│        [输入要删除的 header]                   [添加]        │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│                                        [取消]  [保存]        │
└─────────────────────────────────────────────────────────────┘
```

**优点**：
- 改动较小，保留现有交互
- 通用插件编辑器（基于 Schema 或通用字段适配）
- 表单/JSON 切换在 Drawer 内

**缺点**：
- 下拉选择问题未解决

### 方案三：标签页式插件面板

**核心思路**：将插件选择和配置整合到一个面板中，使用标签页切换不同插件。

**UI 布局**：
```
┌─────────────────────────────────────────────────────────────┐
│  插件管理                                                   │
├─────────────────────────────────────────────────────────────┤
│  可用插件                           已选插件                  │
│  ┌────────────────────────┐  ┌────────────────────────┐      │
│  │ [搜索插件...]          │  │ [ip-rest] [cors] [+] │      │
│  │                        │  └────────────────────────┘      │
│  │ ▸ 限流类              │                                 │
│  │   · limit-req         │  当前编辑: ip-restriction       │
│  │   · limit-count       │  ┌────────────────────────┐      │
│  │ ▸ 认证类               │  │ whitelist:            │      │
│  │   · key-auth          │  │ ┌──────────────────┐  │      │
│  │   · jwt-auth          │  │ │ 127.0.0.1/24    │  │      │
│  │ ▸ 转换类               │  │ └──────────────────┘  │      │
│  │   · proxy-rewrite     │  │                       │      │
│  │   · cors              │  │ [_disable: ○  ●]     │      │
│  └────────────────────────┘  └────────────────────────┘      │
└─────────────────────────────────────────────────────────────┘
```

**优点**：
- 紧凑布局，适合插件数量较多的场景
- 左右分区清晰

**缺点**：
- 实现复杂度中等

### 方案四：渐进式插件配置（最佳用户体验）

**核心思路**：参考 Kong Dashboard 和 APISIX 的设计，分步骤引导用户完成插件配置。

**阶段一：选择插件**
```
┌─────────────────────────────────────────────────────────────┐
│  步骤 1/2: 选择插件                    [取消]                 │
├─────────────────────────────────────────────────────────────┤
│  [搜索插件...]                    按分类: [全部 ▾]           │
│                                                             │
│  限流类 (3)                                                 │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐       │
│  │ limit-req    │ │ limit-count  │ │ limit-conn   │       │
│  │ 请求速率限制  │ │ 请求次数限制  │ │ 连接数限制   │       │
│  │      [+ 添加]│ │      [+ 添加]│ │      [+ 添加]│       │
│  └──────────────┘ └──────────────┘ └──────────────┘       │
│                                                             │
│  认证类 (3)                                                 │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐       │
│  │ key-auth     │ │ jwt-auth     │ │ basic-auth   │       │
│  │              │ │              │ │              │       │
│  └──────────────┘ └──────────────┘ └──────────────┘       │
└─────────────────────────────────────────────────────────────┘
```

**阶段二：配置已选插件**
```
┌─────────────────────────────────────────────────────────────┐
│  步骤 2/2: 配置插件                     [上一步] [完成]       │
├─────────────────────────────────────────────────────────────┤
│  已选择插件:                                                │
│  [ip-restriction ×] [cors ×] [proxy-rewrite ×]            │
│                                                             │
│  ┌─ ip-restriction ─────────────────────────────────────┐ │
│  │  whitelist:                                            │ │
│  │  ┌──────────────────────────────────────────────┐     │ │
│  │  │ 127.0.0.1/24, 10.0.0.0/8                    │     │ │
│  │  └──────────────────────────────────────────────┘     │ │
│  │                                                        │ │
│  │  [_disable: ○  ●]                                    │ │
│  └──────────────────────────────────────────────────────┘ │
│                                                             │
│  ┌─ cors ────────────────────────────────────────────────┐ │
│  │  allow_origins: *                                     │ │
│  │  allow_methods: GET, POST                           │ │
│  └──────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

**优点**：
- 用户体验最佳，分步骤降低认知负担
- 可以批量配置多个插件
- 支持插件排序

**缺点**：
- 需要较大改动
- 多步骤流程可能让高级用户觉得繁琐

---

## 建议

推荐**方案二（侧边抽屉式）**作为起点，原因：
1. 改动最小，保留现有交互模式
2. 解决核心问题（通用插件编辑器）
3. 表单/JSON 切换统一在 Drawer 内

如果时间允许，可以考虑**方案四**作为最终目标。

## 待确认问题

2. 插件数量预计有多少个？（影响 UI 设计）
3. 用户角色是管理员还是普通业务人员？（影响功能复杂度）
