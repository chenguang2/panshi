## 需求1：节点操作列"更多"下拉菜单

### 方案A：纯下拉菜单

```
┌──────────────────────────────────────────────────────────────────────────┐
│  节点信息                       操作                                     │
│  10.0.1.1:9180 │ 编辑 │ 删除 │ 启动 │ 更多 ▼                          │
│  10.0.1.2:9180 │ 编辑 │ 删除 │ 启动 │ 更多 ▼                          │
│                                          ┌──────────────┐               │
│                                          │ 停止          │               │
│                                          │ 状态查询      │               │
│                                          │ 数据库对比    │               │
│                                          └──────────────┘               │
└──────────────────────────────────────────────────────────────────────────┘
```

**规则**：已选中的按钮正常显示（如编辑/删除/启动），更多里显示未选中的（停止/状态查询/数据库对比）。

**特点**：简单直接，与现有列设置联动。

### 方案B：Popover 悬浮菜单

```
┌──────────────────────────────────────────────────────────────────────────┐
│  10.0.1.1:9180 │ 编辑 │ 删除 │ 启动 │ ⋮   │                           │
│                                         ┌──────────────┐                │
│                                         │ 停止          │               │
│                                         │ 状态查询      │               │
│                                         │ 数据库对比    │               │
│                                         └──────────────┘                │
└──────────────────────────────────────────────────────────────────────────┘
```

**特点**：用 `⋮`（三点）图标代替"更多"文字，更紧凑。所有操作都通过列设置控制显隐，可选的在表格中显示，未选的在三点菜单中出现。

## 需求2：上游/路由发布状态标识

### 方案A：表格新增列

上游表格：

```
┌────────┬──────────┬──────────┬──────────┬──────────┬──────────────────────┐
│ 名称   │ 负载均衡 │ 描述     │ 版本状态 │ 发布时间 │ 操作                 │
├────────┼──────────┼──────────┼──────────┼──────────┼──────────────────────┤
│ api-v1 │ 加权轮询 │ 主服务   │ ✅ v3   │ 05-14 10:30│ 编辑 │ 删除 │ 发布  │
│ old-api│ 一致性哈希│ 旧服务   │ ⏳ 未发布 │ —        │ 编辑 │ 删除 │ 发布  │
└────────┴──────────┴──────────┴──────────┴──────────┴──────────────────────┘
```

**特点**：增加两列（版本状态、发布时间），由列选择器控制显隐。发布状态 Badge 使用颜色区分（绿色=已发布，灰色=未发布）。

### 方案C：行背景色 + Badge

```
┌────────┬──────────┬──────────┬──────────┬──────────────────────────────┐
│ 名称   │ 负载均衡 │ 描述     │ 发布状态 │ 操作                         │

│ old-api│ 一致性哈希│ 旧服务   │ ⚪ 未发布 │ 编辑 │ 删除 │ 发布 │ 版本管理│
└────────┴──────────┴──────────┴──────────┴──────────────────────────────┘
```

**特点**：用圆点图标（🟢/⚪）代替文字"✅"/"⏳"，更紧凑。单独一列"发布状态"由列选择器控制。

---

### 后端数据补充

Upstream/Route 列表 API 需要补充返回字段：

```json
{
  "current_version": 3,
  "published_at": "2026-05-14T10:30:00"
}
```

`published_at` 从 `ConfigVersion` 表中取最新版本的 `created_at`。如果从未发布，两个字段均为 `null`。

## 方案对比总表

| 维度 | 节点更多菜单 | 发布标识方案A | 发布标识方案B | 发布标识方案C |
|------|:---------:|:--------:|:--------:|:--------:|

- 节点更多菜单：**方案A**（纯下拉，与列设置联动，用户可控制显示哪些）
