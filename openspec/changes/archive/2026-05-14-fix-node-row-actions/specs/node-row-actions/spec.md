## ADDED Requirements

### Requirement: 行内菜单编辑操作使用当前行数据

节点表格每行的"更多"下拉菜单中，"编辑"按钮 SHALL 直接使用该行对应的节点数据进行编辑操作，而不依赖表格的行选中状态。

#### Scenario: 行内菜单编辑不同节点
- **WHEN** 用户选中节点 A 后，点击节点 B 所在行的"更多"菜单 → "编辑"
- **THEN** 编辑弹窗中显示的是节点 B 的数据

#### Scenario: 未选中任何行时行内编辑
- **WHEN** 未选中任何行时，点击某行"更多"菜单 → "编辑"
- **THEN** 编辑弹窗中显示的是该行节点的数据

### Requirement: 行内菜单删除操作使用当前行数据

节点表格每行的"更多"下拉菜单中，"删除"按钮 SHALL 直接使用该行对应的节点数据进行删除确认和操作，而不依赖表格的行选中状态。

#### Scenario: 行内菜单删除不同节点
- **WHEN** 用户选中节点 A 后，点击节点 B 所在行的"更多"菜单 → "删除"
- **THEN** 删除确认弹窗中显示的 IP 是节点 B 的 IP
- **AND** 确认后删除的是节点 B

### Requirement: 顶部工具栏按钮行为保持不变

顶部的"编辑节点"和"删除节点"按钮 SHALL 继续使用当前选中行（`cluster.selectedNode`）进行操作。

#### Scenario: 顶部按钮使用选中行
- **WHEN** 用户选中某行后点击顶部"编辑节点"按钮
- **THEN** 编辑弹窗中显示的是选中行的数据
