## Why

当前节点管理功能嵌入在集群详情页面的 Tab 中，操作路径长、缺乏全局视图。运维人员需要跨集群查看和管理所有 Edge 网关节点，当前的设计无法满足"统一查看、统一运维"的需求。新增独立的节点管理页面，提供全局节点列表、跨集群筛选、一键启停和状态查询，大幅提升日常运维效率。

## What Changes

- 新增独立节点管理页面 `/nodes`，展示所有集群的全部节点
- **Nginx 列改为 Edge版本**，展示节点的 Edge 软件版本信息
- **去掉心跳时间列**
- **操作列**：详情、启动、停止、状态查询（主按钮），更多菜单含编辑、删除、数据库对比
- **集群筛选下拉框**改为"全部集群"（默认显示全部）
- **增加状态筛选**：全部状态 / 运行中 / 已停止
- **搜索框**同时搜索 IP 和名称
- **添加节点表单**增加"所属集群"下拉选择框
- **编辑节点**不可修改所属集群
- **权限模型**沿用跨集群 API 模式，非管理员只看到有权限集群的节点
- 后端扩展 `GET /nodes` 为全局节点列表 API（支持跨集群查询、分页、搜索、筛选、状态过滤），返回集群名称，增加认证依赖
- 路由注册：前端新增 `/nodes` 路由
- 侧边栏导航新增"节点管理"入口

## Capabilities

### New Capabilities
- `node-management`: 全局节点管理能力，包括跨集群节点列表查看、筛选、搜索、节点操作（启动/停止/状态查询/数据库对比）、节点 CRUD

### Modified Capabilities
- 无（现有集群详情页中的节点 Tab 保持不变）

## Impact

- **后端**：`backend/app/api/v1/nodes.py` 扩展为完整 CRUD + 操作 API；现有 `cluster_nodes.py` 保持不变
- **前端**：新增 `frontend/src/views/NodeList.vue`；修改 `frontend/src/router/index.ts` 添加路由；修改 `frontend/src/components/AppSidebar.vue` 添加导航项
- **依赖**：无新增依赖
