## Why

当前用户管理页面（UserList.vue）功能较为基础，缺少设计样稿（users.html）中展示的筛选栏、角色徽章、权限标签网格、集群权限选择、操作下拉菜单、分页等核心功能。需要按照设计稿全面重构，使页面既美观又功能完整。

## What Changes

- **筛选栏**：搜索框 + 角色筛选下拉 + 状态筛选下拉 + 用户计数
- **表格列扩展**：复选框列、用户名、角色徽章（管理员蓝/普通用户绿）、状态徽章、权限标签（彩色小标签）、可访问集群标签、创建时间、操作下拉菜单（⋯ → 编辑/启用禁用/删除）
- **角色徽章**：管理员使用蓝色徽章，普通用户使用绿色徽章
- **状态徽章**：使用 BadgeStatus 组件（启用=绿色/禁用=红色）
- **操作下拉菜单**：三点（⋯）按钮弹出下拉菜单，包含编辑/启用禁用/删除选项，删除/禁用初始管理员时拦截
- **新建/编辑弹窗**：
  - 用户名 + 角色下拉（管理员/普通用户）
  - 密码输入（新建时必填，编辑时隐藏）
  - 启用/禁用复选框
  - 资源权限网格（插件组管理/全局规则管理/边缘节点管理/插件管理），管理员时隐藏
  - 不使用集群权限选择器（后端无集群级别权限模型）
  - 密码重置区（编辑模式显示）
- **分页控件**：每页显示 + 页码导航
- **API 兼容**：保留现有 `/admin/users` 和 `/admin/users/${id}/permissions` 接口，不做后端改动

## Capabilities

### New Capabilities
- `user-filter-bar`: 搜索 + 角色/状态筛选 + 计数
- `user-role-badge`: 角色徽章组件（管理员蓝/普通用户绿）
- `user-permission-grid`: 资源权限网格选择器（4 项后端权限 key）
- `user-action-dropdown`: 操作三点菜单（编辑/启用禁用/删除）
- `user-modal`: 新建/编辑弹窗（含权限网格 + 集群标签 + 密码重置）

### Modified Capabilities
（无 spec 级别的行为变更）

## Impact
