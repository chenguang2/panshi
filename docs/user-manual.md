## 1. 快速开始

### 1.1 访问与登录

打开浏览器访问 `http://localhost:9100`，使用默认账号登录：

| 用户名 | 密码 | 角色 |
|---|---|---|
| `admin` | `panshi123` | 管理员（完全访问） |

登录后进入仪表盘页面，可以看到全局概览。

### 1.2 第一个操作流程：创建并发布路由

```
1. 添加集群 → 2. 添加上游 → 3. 添加路由 → 4. 发布到节点
```

**步骤一：添加集群**
1. 点击左侧导航「集群管理」
2. 点击「+ 新建集群」
3. 填写集群名称、Edge 管理地址和管理密钥
4. 点击「连接测试」验证连通性
5. 点击「保存」

**步骤二：添加上游**
1. 进入「上游管理」
2. 点击「+ 新建上游」
3. 选择所属集群
4. 填写上游名称，添加目标节点（IP:端口 + 权重）
5. 选择负载均衡算法，配置超时和健康检查
6. 点击「保存」

**步骤三：添加路由**
1. 进入「路由管理」
2. 点击「+ 新建路由」
3. 选择所属集群
4. 填写路由名称、URI（如 `/api/v1/*`）
5. 选择前端匹配的上游
6. 配置方法（GET/POST 等）、域名、优先级
7. 点击「保存」

**步骤四：发布到节点**
1. 在路由列表中点击路由的「发布」按钮
2. 选择目标 Edge 节点
3. 点击确认，查看发布结果

### 1.3 页面布局说明

所有资源管理页面遵循统一布局：

```
┌─────────────────────────────────────────────────┐
│  页面标题  [页面描述]                    [+ 新建] │
├─────────────────────────────────────────────────┤
│  [搜索框]  [分组筛选]  [集群筛选]  [状态筛选]  [共 N 条] │
├─────────────────────────────────────────────────┤
│                                                  │
│  资源列表（表格或卡片网格）                        │
│                                                  │
├─────────────────────────────────────────────────┤
│  分页（仅表格页面）                               │
└─────────────────────────────────────────────────┘
```

## 2. 仪表盘

仪表盘是登录后的首页，展示全局概览信息。

### 2.1 统计卡片

顶部统计卡片展示关键数据：

| 指标 | 说明 |
|---|---|
| 集群总数 | 所有已创建的集群数量 |
| 节点总数 / 在线节点 | Edge 节点总数和运行中的节点数 |
| 上游总数 | 所有上游服务数量 |
| 路由总数 | 所有路由规则数量 |

### 2.2 最近修改的路由

展示最近更新的路由列表，包含名称、URI、所属集群、版本号和更新时间。每页最多 20 条。

### 2.3 集群状态表

所有集群的状态概览表格，每行显示：

| 列 | 说明 |
|---|---|
| 集群名称 | 带分组标签 |
| 节点数 | 在线/总数 |
| 配置版本 | 当前版本号 |
| 上游数 / 路由数 | 资源计数（点击可跳转） |

## 3. 集群管理

### 3.1 集群列表

集群页面以卡片网格形式展示所有集群，按**分组**进行折叠分类。

**分组显示**：
- 同一分组的集群放在同一区块，区块标题可点击折叠/展开
- 顶部有「全部展开」/「全部收起」按钮
- 无分组的集群显示在「未分类」区块

**卡片信息**：
- 顶栏：分组名称（颜色标识分组）
- 标题：集群显示名（或名称）
- 状态标签：运行中 / 已禁用
- 资源统计：节点、上游、路由、插件组等计数（点击可跳转到对应页面）
- 节点标签：显示前 3 个节点的 IP:端口及在线状态

### 3.2 新建集群

点击「+ 新建集群」，填写以下信息：

| 字段 | 必填 | 说明 |
|---|---|---|
| 集群名称 | 是 | 字母数字组合，小写，可包含连字符 |
| 显示名称 | 否 | 前端显示的中文名称 |
| 分组名称 | 否 | 用于分组分类，同组集群共享同一分组名 |
| Edge 管理地址 | 是 | Edge 节点的管理 API 地址 |
| Edge 管理密钥 | 是 | Edge 节点的 API 认证密钥 |

创建后建议执行「连接测试」验证配置正确性。

### 3.3 编辑与删除

- **编辑**：修改集群信息、分组归属
- **连接测试**：验证 Edge 管理 API 连通性
- **删除**：删除集群及其所有关联资源。可选择同时删除数据库记录和 Edge 节点上的配置

### 3.4 分组管理

集群的 `group_name` 字段用于分组。分组的作用：

1. **集群页面**：按分组折叠展示集群卡片
2. **资源筛选**：在所有资源管理页面（上游、路由、插件组等）的顶部都有「分组筛选」下拉框
3. **视觉标识**：卡片顶栏的背景颜色根据分组名自动分配（基于哈希的确定性调色板），同一分组颜色一致

**未分组集群**：没有设置 `group_name` 的集群属于「未分类」，在分组筛选中对应「未分组」选项。

## 4. 上游管理

### 4.1 上游列表

上游管理页面以表格形式展示所有上游服务，支持：

- **分组筛选**：按集群分组过滤
- **集群筛选**：按所属集群过滤
- **负载均衡算法筛选**：按算法类型过滤
- **搜索**：按名称或描述搜索
- **排序**：点击列头排序（名称、集群、负载均衡类型等）

### 4.2 添加上游

点击「+ 新建上游」，配置以下内容：

**基本信息**：
| 字段 | 必填 | 说明 |
|---|---|---|
| 所属集群 | 是 | 选择上游所属集群 |
| 名称 | 是 | 上游服务名称 |
| 描述 | 否 | 可选描述 |
| 负载均衡算法 | 是 | 见下方算法说明 |

**目标节点**：添加一个或多个后端目标地址：
- **目标**：`IP:端口` 格式
- **权重**：负载均衡权重（默认 100）

**负载均衡算法**：

| 算法 | 说明 | 适用场景 |
|---|---|---|
| 加权轮询 | 按权重轮流分发请求 | 通用场景，默认推荐 |
| 一致性哈希 | 根据请求特征（如 IP 或 URI）哈希分发 | 需要会话保持的场景 |
| EWMA | 指数加权移动平均，根据响应时间动态分配 | 后端性能差异较大的场景 |
| 最少连接 | 分发到当前活跃连接最少的节点 | 长连接场景 |

**高级配置**（可选展开）：
- **健康检查**：`HTTP` 或 `HTTPS` 主动健康检查
  - 检查间隔（秒） / 超时（秒） / 健康阈值 / 不健康阈值
  - HTTP 检查路径
- **超时**：连接超时 / 读取超时 / 发送超时
- **连接池**：最大连接数 / 最大空闲连接数

### 4.3 编辑与发布

- **编辑**：修改上游配置，支持添加/删除目标节点
- **发布**：将上游配置推送到 Edge 节点。发布后创建版本快照
- **版本管理**：查看历史版本、对比版本差异、回滚到任意版本
- **删除**：删除上游，可选择同步删除 Edge 节点上的配置

## 5. 路由管理

### 5.1 路由列表

路由管理页面以表格形式展示所有路由规则，支持：

- **方法筛选**：按 HTTP 方法（GET/POST/PUT/DELETE 等）快速筛选
- **分组/集群/上游/插件/发布状态筛选**：多维过滤
- **搜索**：按名称、URI、描述搜索
- **排序**：点击列头排序

### 5.2 新建路由

点击「+ 新建路由」，使用多 Tab 编辑表单：

**基本信息 Tab**：
| 字段 | 必填 | 说明 |
|---|---|---|
| 所属集群 | 是 | 选择路由所属集群 |
| 名称 | 是 | 路由名称 |
| URI | 是 | 匹配路径，支持 `*` 通配符（如 `/api/v1/*`） |
| 方法 | 否 | 匹配的 HTTP 方法（多选：GET/POST/PUT/DELETE/PATCH） |
| 域名 | 否 | 匹配的域名（支持通配符如 `*.example.com`） |
| 优先级 | 否 | 数字，越高越优先匹配 |
| 前端上游 | 否 | 选择处理请求的上游服务 |
| 描述 | 否 | 可选描述 |

**高级匹配 Tab**：
- 支持按请求头（Header）、查询参数（Query）、POST 参数、Cookie 进行条件匹配
- 操作符：`==` / `!=` / `>` / `<` / `~~`（正则） / `IN` / `NOT IN`
- 多个条件之间的逻辑关系为 AND

**插件 Tab**：
- 为路由绑定插件（见第 6 节）

### 5.3 发布与版本管理

- **发布**：选择目标节点推送到 Edge
- **复制路由**：基于现有路由创建新路由，保留所有配置
- **版本管理**：查看版本历史、差异对比、回滚

## 6. 插件管理

磐石 Admin 支持四层插件管理机制：

### 6.1 路由级插件

直接绑定到单个路由的插件配置。在路由编辑表单的「插件」Tab 中配置。

### 6.2 插件组（PluginConfig）

可复用的插件配置集合，可被多个路由引用。

**创建插件组**：
1. 进入「插件组」页面
2. 点击「+ 添加插件组」
3. 选择所属集群，填写名称
4. 点击「添加插件」选择插件类型
5. 编辑插件配置（支持表单模式或 JSON 模式）
6. 保存后在路由编辑 Tab 中引用

**插件编辑器**支持两种模式：
- **表单模式**：按插件 Schema 渲染表单字段，直观填写
- **JSON 模式**：直接编辑 JSON 配置，适合高级用户

两种模式实时同步，切换不丢失数据。

### 6.3 全局规则（GlobalRule）

对整个集群生效的插件配置，影响该集群的所有路由。

创建和编辑方式与插件组一致，区别在于作用范围是全局的。

### 6.4 插件元数据（PluginMetadata）

可复用的插件模板，定义插件的默认参数和配置 Schema。

**典型用途**：预先定义好某个插件的通用配置模板，在创建插件组或全局规则时直接使用，减少重复配置。

### 6.5 内置插件

系统内置 20+ 个常用插件，按类别划分：

| 类别 | 插件示例 |
|---|---|
| 安全 | key-auth、basic-auth、jwt-auth、ip-restriction、ua-restriction |
| 限流 | limit-req、limit-conn |
| 转换 | response-rewrite、proxy-rewrite、referer-restriction |
| 流量复制 | traffic-split |
| 常规 | cors、static-resource |

插件可通过「插件开关」页面启用/禁用，禁用后在前端表单中不可选。

## 7. 节点管理

### 7.1 节点列表

节点管理页面以表格形式展示所有 Edge 节点，支持：

- **分组/集群/状态筛选**：按节点状态（运行中/已停止）过滤
- **搜索**：按 IP 或名称搜索
- **排序**：点击列头排序
- **状态列**：显示 nginx 运行状态和 Edge 版本号

### 7.2 节点生命周期操作

每个节点的操作列提供以下按钮：

| 操作 | 说明 |
|---|---|
| ▶ 启动 | 通过 Ansible 启动节点上的 Nginx |
| ⏹ 停止 | 停止 Nginx 服务 |
| ✓ 状态 | 查询 Nginx 运行状态和系统统计信息 |
| ⓘ 详情 | 查看节点属性和集群统计卡片 |
| 编辑 | 修改节点配置（IP、端口、路径等） |
| 删除 | 删除节点记录 |
| 数据库对比 | 对比本地配置与节点运行配置 |
| 安装 OpenResty | 在节点上安装 OpenResty（需功能开关） |
| 安装 Edge | 在节点上安装 Edge（需功能开关） |

**执行结果 Drawer**：启动、停止、状态查询等操作会打开执行结果 Drawer，实时显示 Ansible 执行日志、返回码、stdout/stderr。

**状态自动同步**：节点操作成功后，`node.status` 会自动更新。停止成功→`status=0`，启动成功→`status=1`。状态查询根据 Nginx 实际运行状态同步更新。

### 7.3 批量操作

支持对多个节点批量执行同一操作（启动、停止、重启、状态查询、检查）：

```
1. 在节点列表页面使用复选框选择节点
2. 点击顶部的批量操作按钮
3. 确认操作
4. 查看每个节点的执行结果
```

## 8. 四层代理

### 8.1 四层代理列表

以卡片网格形式展示所有 TCP/UDP 四层代理规则，每张卡片显示：

- 顶栏：所属集群 + 所属分组（颜色标识）
- 协议（TCP/UDP）
- 监听端口
- 负载均衡算法
- 目标节点列表

### 8.2 新建四层代理

使用多步向导表单：

**第一步：基本信息**
- 所属集群、名称、描述
- 协议（TCP/UDP）
- 监听端口（支持端口占用检测）

**第二步：配置**
- 负载均衡算法
- 后端目标（IP:端口 + 权重）
- 超时配置
- 连接池配置

### 8.3 端口检测

在编辑四层代理时，系统提供端口占用检测功能，帮助用户选择未被占用的监听端口。检测结果包括端口号、状态（空闲/占用）和占用来源。

## 9. 静态资源

### 9.1 静态资源列表

以卡片网格形式展示所有 ZIP 静态资源，每张卡片显示：

- 顶栏：所属集群 + 分组
- 资源名称
- URL 路径
- 发布状态
- 文件大小
- 描述

### 9.2 添加静态资源

1. 点击「+ 添加静态资源」
2. 选择所属集群和关联路由（路由必须满足：`/*` 结尾、已发布、挂载了 `static_resource` 插件）
3. 填写描述
4. 保存后上传 ZIP 文件

### 9.3 ZIP 上传与管理

- 上传 ZIP 文件后系统自动解压并将内容同步到 Edge 节点
- 支持查看 ZIP 内容列表（文件名、大小、压缩比）
- 发布后前端可通过配置的 URL 路径访问静态资源

## 10. Edge 工具

### 10.1 Edge 直连客户端

直接与 Edge 节点管理 API 通信的工具页面，支持：

- **上游管理**：查看、创建、编辑、删除 Edge 节点上的上游配置
- **路由管理**：查看、创建、编辑、删除路由配置
- **插件管理**：查看、管理插件组和全局规则
- **四层代理**：查看和管理 Stream Route

每类资源都提供统一的表格视图和操作按钮。

### 10.2 Edge 数据导入

将 Edge 节点上的运行配置批量导入到本地数据库：

**三步骤流程**：
1. **连接测试**：选择 Edge 节点，测试连通性
2. **预览**：导入前预览将要导入的数据，系统自动检测冲突
3. **执行导入**：确认后执行导入，可选覆盖或跳过冲突项

支持导入的资源类型：上游、路由、插件组、全局规则、插件元数据、四层代理。

### 10.3 Edge 环境配置

远程读取和编辑 Edge 节点的 `edge.env` 配置文件：

- **读取**：从 Edge 节点获取当前 edge.env 配置
- **编辑**：使用 Monaco 编辑器在线修改
- **部署**：将修改后的配置推送到 Edge 节点（SSE 流式日志）
- **版本管理**：每次部署创建版本记录，支持回滚

### 10.4 配置对比

对比本地数据库中的期望配置与 Edge 节点上的实际运行配置：

- 支持所有资源类型（上游、路由、插件组、全局规则、插件元数据）
- 差异化高亮显示新增、修改、删除的字段
- 使用等效规则引擎处理语义等价但格式不同的字段（如 `weighted_roundrobin` ↔ `roundrobin`）

## 11. 系统管理

### 11.1 用户管理

管理员可管理所有用户账号：

| 功能 | 说明 |
|---|---|
| 创建用户 | 设置用户名、密码、角色（admin/user） |
| 编辑用户 | 修改密码、角色、状态 |
| 分配集群 | 将集群访问权限分配给指定用户 |
| 分配权限 | 按资源类型控制用户的可操作范围 |
| 启用/禁用 | 禁用后用户无法登录和使用系统 |

### 11.2 插件开关

控制内置插件的启用/禁用状态：

- 禁用后的插件在所有表单中不可选
- 启用已禁用的插件时，系统自动检测是否有路由或配置正在引用该插件
- 插件开关状态存储在 `ps_plugin_enabled` 表

### 11.3 开发者工具

提供常用开发和调试工具：

- **Lua ↔ 字符串转换**：用于 Lua 配置中的转义处理
- **URL 编解码**：URL 参数编解码
- **Base64 编解码**
- **JSON 格式化**
- **YAML 格式化**

## 12. 常见操作指南

### 12.1 如何创建一条可访问的 API

```
1. 集群管理 → 新建集群（填写 Edge 地址和密钥）
2. 上游管理 → 添加上游（添加后端服务地址）
3. 路由管理 → 新建路由（设置 URI 匹配 `/api/*`，选择上游）
4. 路由管理 → 点击发布 → 选择 Edge 节点
→ 访问 http://<edge_ip>:<service_port>/api/... 即可
```

### 12.2 如何安全地变更配置

```
1. 修改配置（路由/上游/插件等）
2. 发布前可先使用 Edge 直连客户端查看当前 Edge 节点配置
3. 使用「数据库对比」功能对比期望配置与运行配置
4. 确认无误后发布
5. 发布后使用 Edge 直连客户端验证配置已生效
6. 如出现问题，使用「版本管理」回滚到上一个版本
```

### 12.3 如何对路由启用限流插件

```
1. 路由管理 → 编辑目标路由
2. 切换到「插件」Tab
3. 点击「添加插件」
4. 选择「limit-req」插件
5. 配置限流参数（每秒请求数、突发大小等）
6. 保存 → 发布路由
```

### 12.4 如何使用分组管理集群

```
1. 创建集群时填写「分组名称」字段
2. 或在集群编辑页面设置分组
3. 在集群管理页面，集群自动按分组折叠展示
4. 在其他资源页面（上游、路由等），使用顶部的「分组筛选」下拉框
5. 卡片顶栏的颜色根据分组自动分配
```

### 12.5 如何导入现有 Edge 配置

```
1. 确认集群已创建且 Edge 地址/密钥正确
2. 进入 Edge 导入页面
3. 选择目标集群和 Edge 节点
4. 点击「连接测试」
5. 选择要导入的资源类型
6. 预览导入结果，检查冲突
7. 执行导入
```

## 13. 故障排除

### 13.1 登录问题

| 问题 | 可能原因 | 解决方法 |
|---|---|---|
| 登录提示"用户名或密码错误" | 密码错误 | 联系管理员重置密码 |
| 登录后立即跳回登录页 | Token 过期或无效 | 清除浏览器缓存后重新登录 |
| 页面空白 | 后端未启动 | 启动后端服务（`uv run uvicorn app.main:app --port 9000`） |

### 13.2 发布失败

| 错误信息 | 可能原因 | 解决方法 |
|---|---|---|
| "节点不存在" | 节点未创建或已被删除 | 检查节点列表，确认节点状态 |
| "连接失败" | Edge 节点不可达或管理端口错误 | 检查网络连通性、管理端口和密钥 |
| "配置验证失败" | Edge API 返回格式错误 | 在 Edge 直连客户端中测试 API |
| "操作失败: SSH connection failed" | Ansible SSH 连接失败 | 检查网络、SSH 密钥和 `sshpass` 配置 |

### 13.3 节点操作失败

| 问题 | 可能原因 | 解决方法 |
|---|---|---|
| 启动节点后状态未更新 | Ansible 命令执行成功但 nginx 实际未启动 | 执行「状态查询」重新检测 |
| 节点状态显示异常 | `status_detail` 被清除或格式错误 | 执行「状态查询」获取最新状态 |
| 安装 OpenResty/Edge 超时 | 编译过程耗时较长 | 等待安装完成，查看流式日志 |

### 13.4 配置不一致

| 问题 | 可能原因 | 解决方法 |
|---|---|---|
| 数据库与 Edge 配置不一致 | 发布后 Edge 节点配置被手动修改 | 使用配置对比功能查看差异，重新发布 |
| 路由不生效 | 路由未发布或 URI 匹配规则不正确 | 检查路由发布状态和 URI 配置 |
| 插件不生效 | 插件被禁用或未部署到 Edge | 检查插件开关状态，重新发布 |

### 13.5 开发环境问题

| 问题 | 可能原因 | 解决方法 |
|---|---|---|
| 前端页面 API 报错 | 后端未启动或端口不匹配 | 确认后端在 9000 端口运行 |
| 数据库错误 | 数据库文件损坏或 Schema 不匹配 | 删除 `backend/data/panshi.db` 后重启后端（会自动重建） |
| 前端构建失败 | TypeScript 类型错误 | 运行 `npm run vue-tsc -b` 查看具体错误 |
