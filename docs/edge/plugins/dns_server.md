# dns_server 插件使用说明书

## 1. 插件概览

`dns_server` 插件用于在 API 网关 L4 Stream 层（UDP 协议）提供内建的高性能 DNS 域名解析解析服务。网关可直接作为权威域名服务器，拦截并响应特定域名的 DNS 查询请求。

主要应用场景：
- **边缘 DNS 解析**：在网关边缘直接响应客户端的域名解析，大幅提高域名首次解析的性能。
- **智能 Geo-DNS 调度**：根据客户端的真实地理位置/子网网段信息，动态返回最接近的或专用的解析 IP 地址，实现智能网络流量调度。

---

## 2. 配置详情

### 2.1 插件静态属性 (Attr Config)

无。

---

### 2.2 插件数据属性 (Metadata Config)

通过管理接口进行全局配置（API 路径：`/edge/admin/plugin_metadata/dns_server`），用于设定全局默认的 DNS 解析映射。

> **[!IMPORTANT]
> 所有插件的数据属性（Metadata Config）变更后，必须执行一次插件热加载操作以使配置生效：
> - **请求方法**：`PUT`
> - **请求路径**：`/edge/admin/plugins/reload`
> 
> *您可以在配置多个插件的数据属性后，统一执行一次热加载操作。*

| 字段名 | 类型 | 默认值 | 描述 | 设置示例 |
| :--- | :--- | :--- | :--- | :--- |
| `hosts` | Object | 无 | 域名解析主机映射表。其键为目标域名，值为包含解析目标和客户端匹配规则的对象，详见下文结构说明。 | 见下文结构说明 |

#### **`hosts` 内部字段结构说明**：
结构格式为：`{ 域名: { "nodes": { "解析地址1": [客户端源地址列表1], "解析地址2": [客户端源地址列表2] } } }`。
每个域名下配置 **`nodes`** 对象：
- **`nodes`** (Object)：键为解析返回的目标 IP（如 `10.0.0.1`），值为匹配该目标的客户端源 IP/CIDR 网段列表（数组）。
  - **智能解析分流 (Geo-DNS)**：网关接收到客户端 DNS 请求时，若客户端源 IP 匹配某解析地址对应的地址列表，则直接返回该解析地址。
  - **默认轮询响应**：当客户端源 IP **不在任何指定的解析地址对应的地址列表中时**，网关将**轮询 (Round-Robin) 响应这些配置的解析地址**，从而实现对不匹配客户端的默认均衡分发。

## 3. 调用与验证示例

### 3.1 创建并启用 Stream 路由

使用 Admin 管理接口创建或更新一条监听 UDP `9971` 端口的 Stream 路由，并为其启用 `dns_server` 插件。我们将为域名 `test.local` 设定解析规则：
- 当客户端 IP 属于本地 `127.0.0.1` 时，强制解析到 `10.0.0.1`；
- 当客户端 IP 属于本地 `192.168.0.0/16` 局域网段时，强制解析到 `10.0.0.2`。

> [!IMPORTANT]
> 调用示例中不包含真实敏感密钥，请将占位符 `API_KEY` 替换为您实际环境中的 Admin API Key。

```bash
curl -i -X PUT \
  -H "Content-Type: application/json" \
  -H "X-Api-Key: API_KEY" \
  -d '{
    "server_addr": "0.0.0.0:9971",
    "protocol": "UDP",
    "plugins": {
      "dns_server": {
        "hosts": {
          "test.local": {
            "nodes": {
              "10.0.0.1": ["127.0.0.1"],
              "10.0.0.2": ["192.168.0.0/16"]
            }
          }
        }
      }
    }
  }' \
  http://127.0.0.1:9991/edge/admin/stream_routes/2001
```

### 3.2 访问网关进行验证

我们可以使用标准的 `dig` 命令向网关的 Stream UDP 监听端口发送 DNS A 记录查询，验证是否能正确获取到域名解析结果：

```bash
dig @127.0.0.1 -p 9971 test.local
```

**预期输出（ANSWER SECTION）**：
由于我们从本机 `127.0.0.1` 发起查询，命中 `10.0.0.1` 的客户端匹配网段，因此解析应准确返回 `10.0.0.1`：

```text
;; QUESTION SECTION:
;test.local.			IN	A

;; ANSWER SECTION:
test.local.		600	IN	A	10.0.0.1
```
