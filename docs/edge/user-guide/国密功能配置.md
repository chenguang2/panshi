# Edge 国密功能配置手册

本文档描述了国密功能的**部署配置步骤**、**握手与业务请求验证记录**，以及**Tongsuo 与标准 OpenSSL 两种构建下的加密套件对比**，供部署时参考。

---

## 一、方案概述

本次验证基于 **Edge + Tongsuo 8.5.0-pre1**，测试国密（NTLS/TLCP）双证书协议在网关侧的启用与实际握手效果。核心结论：

- NTLS 是独立于标准 TLS 协议族的握手体系（本文档中标注为 `NTLSv1.1`），需要单独开启；
- `ECDHE-SM2-*` 系列套件依赖**签名证书 + 加密证书**双证书部署，且要求客户端也提供证书（双向认证）。

## 三、开启并测试 edge 服务的国密功能

### 3.1 准备国密证书

测试域名使用 `gmtest.local`。国密证书可直接复用 `Tongsuo-8.5.0` 中 `test/certs/sm2` 下的证书，也可自行签发。完整的证书体系分为四组：

| 证书 | 用途 |
|---|---|
| `server_sign.crt` / `server_sign.key` | 服务端签名证书，用于身份认证签名 |
| `server_enc.crt` / `server_enc.key` | 服务端加密证书，用于密钥交换环节 |
| `client_sign.crt` / `client_sign.key` | 客户端签名证书（`ECDHE-SM2-*` 双向认证场景需要） |
| `client_enc.crt` / `client_enc.key` | 客户端加密证书 |
| `gm_ca.crt` | 根CA + 子CA 证书链，用于双方验证对端证书 |

> 具体的 PEM 证书内容参见附录 A，也可自行签发，或使用 `Tongsuo-8.5.0` 中 `test/certs/sm2` 下的证书。

### 3.2 配置 hosts

```sh
echo "127.0.0.1  gmtest.local" >> /etc/hosts
```

### 3.3 修改 `edge.env` 并启动 edge 服务

开启 SSL 端口，并启用 `gmssl` 扩展插件：

```yaml
deploy:
  http:
    edge:
      listen:
        - addr: 0.0.0.0:9943
          ssl: true

ex_plugins:
  gmssl: true
```

启动服务：

```sh
bin/edge init
bin/edge start
```

### 3.4 添加国密证书到 edge 服务

通过管理接口添加，注意 `cert`/`key` 对应**加密证书**，`certs`/`keys` 数组对应**签名证书**（NTLS 双证书场景下两者职责不同，不能填反）：

```
POST /edge/admin/ssl/8001
```

```json
{
  "cert": "-----BEGIN CERTIFICATE-----\n...(server_enc.crt)...\n-----END CERTIFICATE-----",
  "key": "-----BEGIN PRIVATE KEY-----\n...(server_enc.key)...\n-----END PRIVATE KEY-----",
  "certs": ["-----BEGIN CERTIFICATE-----\n...(server_sign.crt)...\n-----END CERTIFICATE-----"],
  "keys": ["-----BEGIN PRIVATE KEY-----\n...(server_sign.key)...\n-----END PRIVATE KEY-----"],
  "gm": true,
  "snis": ["gmtest.local"]
}
```

字段对应关系：

```
cert  → server_enc.crt   （加密证书）
key   → server_enc.key   （加密私钥）
certs → [ server_sign.crt ]  （签名证书）
keys  → [ server_sign.key ]  （签名私钥）
gm    → true             （标记为国密证书，触发 NTLS 分支）
snis  → 匹配的域名（SNI）
```

## 四、测试验证

这里 openresty-edge 安装路径：`/usr/local/openresty/test`

### 4.1 确认 openssl 版本

```sh
$ /usr/local/openresty/test/bin/openssl version
Tongsuo: Tongsuo 8.5.0-pre1 (Library: Tongsuo 8.5.0-pre1)
OpenSSL 3.5.4 21 Mar 2026 (Library: OpenSSL 3.5.4 21 Mar 2026)
```

### 4.2 握手测试

只验证 NTLS 握手是否成功，不发送应用层数据：

```sh
/usr/local/openresty/test/bin/openssl s_client \
  -connect gmtest.local:9943 -servername gmtest.local \
  -cipher ECDHE-SM2-SM4-CBC-SM3 -enable_ntls -ntls \
  -verifyCAfile certs/gm_ca.crt \
  -sign_cert certs/client_sign.crt -sign_key certs/client_sign.key \
  -enc_cert certs/client_enc.crt -enc_key certs/client_enc.key
```

关键结果：

```
SSL handshake has read 1344 bytes and written 1323 bytes
Verification: OK
---
New, NTLSv1.1, Cipher is ECDHE-SM2-SM4-CBC-SM3
Server public key is 256 bit
```

`Verification: OK` 说明证书链（root ca → sub ca → server sign/server enc）验证通过；`Peer Temp Key: SM2, 256 bits` 说明用了临时密钥交换，具备前向保密特性。

### 4.3 完整业务请求测试（握手 + HTTP 请求）

握手成功后，进一步验证应用层数据收发：

```sh
echo -e "GET /hello1/h1 HTTP/1.1\r\nHost: gmtest.local\r\nConnection: close\r\n\r\n" | \
/usr/local/openresty/test/bin/openssl s_client \
  -connect gmtest.local:9943 -servername gmtest.local \
  -cipher ECDHE-SM2-SM4-CBC-SM3 -enable_ntls -ntls \
  -verifyCAfile certs/gm_ca.crt \
  -sign_cert certs/client_sign.crt -sign_key certs/client_sign.key \
  -enc_cert certs/client_enc.crt -enc_key certs/client_enc.key \
  -ign_eof
```

**握手部分**（与 4.2 一致，证书链验证通过，套件协商为 `ECDHE-SM2-SM4-CBC-SM3`）：

```
Certificate chain
 0 s:C=CN, ST=Beijing, O=CUN, OU=BB, CN=server sign
   i:C=CN, ST=Beijing, O=CUN, OU=BB, CN=sub ca
   a:PKEY: SM2, 256 (bit); sigalg: SM2-with-SM3
 1 s:C=CN, ST=Beijing, O=CUN, OU=BB, CN=server enc
   i:C=CN, ST=Beijing, O=CUN, OU=BB, CN=sub ca
   a:PKEY: SM2, 256 (bit); sigalg: SM2-with-SM3
---
Peer signing digest: SM3
Peer signature type: sm2sig_sm3
Peer Temp Key: SM2, 256 bits
---
Verification: OK
---
New, NTLSv1.1, Cipher is ECDHE-SM2-SM4-CBC-SM3
Protocol: NTLSv1.1
```

**HTTP 应用层响应**：

```
HTTP/1.1 200 OK
Content-Type: text/plain; charset=utf-8
Transfer-Encoding: chunked
Connection: close
Date: Thu, 16 Jul 2026 01:57:45 GMT

52
gmtest.local:8812/hello1/h1 traceid=AZ9opOhiLX0ziAEAAH8PIwD3JQAAAPc-DwAAAQAAAAAA:

0

closed
```

说明：
- 响应体里回显的 `gmtest.local:8812/hello1/h1` 及 `traceid=...`，说明 `9943` 端口是 NTLS 终止层，请求被正常转发到了后端 `8812` 端口的业务服务，并带上了链路追踪信息；
- `Connection: close` + 末尾 `closed`：服务端主动关闭了连接，属于预期行为。

**结论**：证书体系、NTLS 双证书握手、应用层 HTTP 收发全部验证通过。

## 五、加密套件配置对比

同一份 `ssl_ciphers` 配置在**标准 OpenSSL 3.5.5** 和 **Tongsuo 8.5.0-pre1** 两种构建下，实际生效的套件列表差异很大——核心区别就是 Tongsuo 额外支持国密算法（SM2/SM4/SM3）以及 NTLS 协议分支。

### 5.1 使用标准 openssl-3.5.5

**默认配置**

```
deploy.http.edge.ssl_ciphers: HIGH:!aNULL:!MD5
```

支持列表（节选，完整原始列表见附录 B）：

```
TLS_AES_256_GCM_SHA384         TLSv1.3 Kx=any      Au=any   Enc=AESGCM(256)            Mac=AEAD
TLS_CHACHA20_POLY1305_SHA256   TLSv1.3 Kx=any      Au=any   Enc=CHACHA20/POLY1305(256) Mac=AEAD
TLS_AES_128_GCM_SHA256         TLSv1.3 Kx=any      Au=any   Enc=AESGCM(128)            Mac=AEAD

ECDHE-ECDSA-AES256-GCM-SHA384  TLSv1.2 Kx=ECDH     Au=ECDSA Enc=AESGCM(256)            Mac=AEAD
ECDHE-RSA-AES256-GCM-SHA384    TLSv1.2 Kx=ECDH     Au=RSA   Enc=AESGCM(256)            Mac=AEAD
DHE-RSA-AES256-GCM-SHA384      TLSv1.2 Kx=DH       Au=RSA   Enc=AESGCM(256)            Mac=AEAD
ECDHE-ECDSA-CHACHA20-POLY1305  TLSv1.2 Kx=ECDH     Au=ECDSA Enc=CHACHA20/POLY1305(256) Mac=AEAD
ECDHE-RSA-CHACHA20-POLY1305    TLSv1.2 Kx=ECDH     Au=RSA   Enc=CHACHA20/POLY1305(256) Mac=AEAD
... （共 60+ 条，含 DHE-DSS、ARIA、Camellia、老式 SHA1/CBC 等历史套件）
```

**特点**：不含任何国密算法（SM2/SM4/SM3），适合面向国际标准客户端的常规场景。

**收窄为精确指定套件**（推荐做法，去掉 DHE/老旧算法）：

```
deploy.http.edge.ssl_ciphers: ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:ECDHE-ECDSA-CHACHA20-POLY1305:ECDHE-RSA-CHACHA20-POLY1305
```

支持列表精确收敛为：

```
TLS_AES_256_GCM_SHA384         TLSv1.3 Kx=any      Au=any   Enc=AESGCM(256)            Mac=AEAD
TLS_CHACHA20_POLY1305_SHA256   TLSv1.3 Kx=any      Au=any   Enc=CHACHA20/POLY1305(256) Mac=AEAD
TLS_AES_128_GCM_SHA256         TLSv1.3 Kx=any      Au=any   Enc=AESGCM(128)            Mac=AEAD

ECDHE-ECDSA-AES128-GCM-SHA256  TLSv1.2 Kx=ECDH     Au=ECDSA Enc=AESGCM(128)            Mac=AEAD
ECDHE-RSA-AES128-GCM-SHA256    TLSv1.2 Kx=ECDH     Au=RSA   Enc=AESGCM(128)            Mac=AEAD
ECDHE-ECDSA-AES256-GCM-SHA384  TLSv1.2 Kx=ECDH     Au=ECDSA Enc=AESGCM(256)            Mac=AEAD
ECDHE-RSA-AES256-GCM-SHA384    TLSv1.2 Kx=ECDH     Au=RSA   Enc=AESGCM(256)            Mac=AEAD
ECDHE-ECDSA-CHACHA20-POLY1305  TLSv1.2 Kx=ECDH     Au=ECDSA Enc=CHACHA20/POLY1305(256) Mac=AEAD
ECDHE-RSA-CHACHA20-POLY1305    TLSv1.2 Kx=ECDH     Au=RSA   Enc=CHACHA20/POLY1305(256) Mac=AEAD
```

TLS1.3 三条是库自带固定套件（前 3 行），TLS1.2 精确对应 `ssl_ciphers` 写的 6 个，全部是 ECDHE 前向保密 + AEAD 加密，是目前推荐的"中间兼容"配置。

### 5.2 使用 Tongsuo-8.5.0-pre1

**默认配置**

```
deploy.http.edge.ssl_ciphers: HIGH:!aNULL:!MD5
```

支持列表新增了国密套件（相比 5.1，多出 TLS1.3 里的 2 条 SM 套件 + NTLS 分支的 8 条）：

```
TLS_AES_256_GCM_SHA384         TLSv1.3 Kx=any      Au=any   Enc=AESGCM(256)            Mac=AEAD
TLS_CHACHA20_POLY1305_SHA256   TLSv1.3 Kx=any      Au=any   Enc=CHACHA20/POLY1305(256) Mac=AEAD
TLS_AES_128_GCM_SHA256         TLSv1.3 Kx=any      Au=any   Enc=AESGCM(128)            Mac=AEAD
TLS_SM4_GCM_SM3                TLSv1.3 Kx=any      Au=any   Enc=SM4-GCM(128)           Mac=AEAD
TLS_SM4_CCM_SM3                TLSv1.3 Kx=any      Au=any   Enc=SM4-CCM(128)           Mac=AEAD

ECC-SM2-SM4-GCM-SM3            NTLSv1.1 Kx=SM2      Au=SM2   Enc=SM4-GCM(128)           Mac=AEAD
ECDHE-SM2-SM4-GCM-SM3          NTLSv1.1 Kx=SM2DHE   Au=SM2   Enc=SM4-GCM(128)           Mac=AEAD
RSA-SM4-GCM-SHA256             NTLSv1.1 Kx=RSA      Au=RSA   Enc=SM4-GCM(128)           Mac=AEAD
RSA-SM4-GCM-SM3                NTLSv1.1 Kx=RSA      Au=RSA   Enc=SM4-GCM(128)           Mac=AEAD
ECC-SM2-SM4-CBC-SM3            NTLSv1.1 Kx=SM2      Au=SM2   Enc=SM4(128)               Mac=SM3
ECDHE-SM2-SM4-CBC-SM3          NTLSv1.1 Kx=SM2DHE   Au=SM2   Enc=SM4(128)               Mac=SM3
RSA-SM4-CBC-SHA256             NTLSv1.1 Kx=RSA      Au=RSA   Enc=SM4(128)               Mac=SHA256
RSA-SM4-CBC-SM3                NTLSv1.1 Kx=RSA      Au=RSA   Enc=SM4(128)               Mac=SM3

ECDHE-ECDSA-AES256-GCM-SHA384  TLSv1.2 Kx=ECDH     Au=ECDSA Enc=AESGCM(256)            Mac=AEAD
ECDHE-RSA-AES256-GCM-SHA384    TLSv1.2 Kx=ECDH     Au=RSA   Enc=AESGCM(256)            Mac=AEAD
... （其余同 5.1，含 DHE-DSS/CCM/老式套件等，完整原始列表见附录 C）
```

**要点**：
- `TLS_SM4_GCM_SM3` / `TLS_SM4_CCM_SM3` 属于标准 TLS1.3 框架（RFC 8998），走正常 TLS1.3 握手流程；
- 8 条 `NTLSv1.1` 标注的套件属于国密双证书 TLCP 协议；
- `ECC-SM2-*` 单证书、无前向保密；`ECDHE-SM2-*` 双证书、有前向保密且强制双向认证，两者按安全需求二选一。

**精确指定套件**（国密 + 国际标准套件混合）：

```
deploy.http.edge.ssl_ciphers: ECC-SM2-SM4-GCM-SM3:ECDHE-SM2-SM4-GCM-SM3:ECC-SM2-SM4-CBC-SM3:ECDHE-SM2-SM4-CBC-SM3:ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:ECDHE-ECDSA-CHACHA20-POLY1305:ECDHE-RSA-CHACHA20-POLY1305
```

支持列表精确收敛为：

```
TLS_AES_256_GCM_SHA384         TLSv1.3 Kx=any      Au=any   Enc=AESGCM(256)            Mac=AEAD
TLS_CHACHA20_POLY1305_SHA256   TLSv1.3 Kx=any      Au=any   Enc=CHACHA20/POLY1305(256) Mac=AEAD
TLS_AES_128_GCM_SHA256         TLSv1.3 Kx=any      Au=any   Enc=AESGCM(128)            Mac=AEAD
TLS_SM4_GCM_SM3                TLSv1.3 Kx=any      Au=any   Enc=SM4-GCM(128)           Mac=AEAD
TLS_SM4_CCM_SM3                TLSv1.3 Kx=any      Au=any   Enc=SM4-CCM(128)           Mac=AEAD

ECC-SM2-SM4-GCM-SM3            NTLSv1.1 Kx=SM2      Au=SM2   Enc=SM4-GCM(128)           Mac=AEAD
ECDHE-SM2-SM4-GCM-SM3          NTLSv1.1 Kx=SM2DHE   Au=SM2   Enc=SM4-GCM(128)           Mac=AEAD
ECC-SM2-SM4-CBC-SM3            NTLSv1.1 Kx=SM2      Au=SM2   Enc=SM4(128)               Mac=SM3
ECDHE-SM2-SM4-CBC-SM3          NTLSv1.1 Kx=SM2DHE   Au=SM2   Enc=SM4(128)               Mac=SM3

ECDHE-ECDSA-AES128-GCM-SHA256  TLSv1.2 Kx=ECDH     Au=ECDSA Enc=AESGCM(128)            Mac=AEAD
ECDHE-RSA-AES128-GCM-SHA256    TLSv1.2 Kx=ECDH     Au=RSA   Enc=AESGCM(128)            Mac=AEAD
ECDHE-ECDSA-AES256-GCM-SHA384  TLSv1.2 Kx=ECDH     Au=ECDSA Enc=AESGCM(256)            Mac=AEAD
ECDHE-RSA-AES256-GCM-SHA384    TLSv1.2 Kx=ECDH     Au=RSA   Enc=AESGCM(256)            Mac=AEAD
ECDHE-ECDSA-CHACHA20-POLY1305  TLSv1.2 Kx=ECDH     Au=ECDSA Enc=CHACHA20/POLY1305(256) Mac=AEAD
ECDHE-RSA-CHACHA20-POLY1305    TLSv1.2 Kx=ECDH     Au=RSA   Enc=CHACHA20/POLY1305(256) Mac=AEAD
```

这份精简配置去掉了 `RSA-SM4-*`（无前向保密的兼容折中方案）和所有老旧/CCM/DHE-DSS 套件，只保留了"国密 GCM 优先、CBC 兜底 + 国际标准 ECDHE-GCM/ChaCha20"，是兼顾国密合规要求与安全强度的推荐配置。
