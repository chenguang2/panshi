# Edge Admin vs 磐石 Admin 差距分析报告

> 生成日期：2026-06-15
> 参考文档：https://doc.openresty.com.cn/cn/edge/edge-admin/

---

## 一、核心架构差异

Edge Admin 以 **"应用 (Application)"** 为核心组织单元：

```
应用 → 域名/路由 → 上游 → 插件 → 发布
```

而磐石 Admin 以 **"集群 → 资源"** 为核心：

```
集群 → { 上游, 路由, 插件组, 全局规则 } → 发布
```

Edge Admin 的"应用"抽象使得一组域名 + 路由 + 上游 + 插件的整体管理更便捷。

## 二、磐石已实现的功能

| 模块 | 已有功能 |
|---|---|
| 仪表盘 | 集群概览、健康状态、9 种统计计数器 |
| 集群管理 | 增删改查、节点连接测试、分组 |
| 上游管理 | CRUD、4 种负载均衡策略、健康检查、超时、连接池、发布、版本历史、回滚 |
| 路由管理 | CRUD、域名/路径/请求头/方法匹配、高级条件(vars)、路由级插件、发布、版本历史、回滚 |
| 插件组 | CRUD、发布、版本历史、回滚 |
| 全局规则 | CRUD、发布、版本历史、回滚 |
| 插件开关 | 全局启停、引用检测 |
| 插件元数据 | 可复用模板、参数 schema、发布、版本历史、回滚 |
| 静态资源 | ZIP 上传、目录浏览、关联路由、发布、版本历史、回滚 |
| Edge 直连 | 直接连接 Edge 节点管理所有资源 |
| Edge 数据导入 | 从 Edge 批量导入上游/路由/插件 |
| 配置对比 | 本地 vs Edge 节点运行配置差异高亮 |
| 配置历史 | 变更追踪、回滚 |
| 用户管理 | CRUD、角色、权限、集群分配 |
| 认证 | JWT 登录/注销/权限查询 |
| 节点安装 | OpenResty 安装、Edge 安装、取消安装 |
| 系统管理 | 特性开关、插件白名单 |

## 三、Edge Admin 有但我们缺失的功能

### P0 - 核心缺失功能

| # | 功能 | Edge Admin 文档位置 | 说明 |
|---|---|---|---|
| 1 | **代理缓存** | `proxy-cache/` | 路由级别的缓存策略配置（缓存时间、缓存 key、缓存状态），网关核心功能 |
| 2 | **响应压缩** | `compression/gzip/`, `brotli/`, `zstd/` | 三种压缩算法（Gzip / Brotli / Zstandard）配置，目前完全缺失 |
| 3 | **CC 攻击防护** | `ddos-logs/` | CC 攻击日志和防护策略，生产环境必备 |
| 4 | **SSL/TLS 证书管理** | （路由层面） | 上传和管理 SSL 证书，绑定到域名/路由 |

### P1 - 重要增强功能

| # | 功能 | 说明 |
|---|---|---|
| 5 | **gRPC 代理** | 上游层面支持 gRPC 协议，目前只支持 HTTP |
| 6 | **K8s 服务发现** | 绑定上游到 k8s 服务，自动解析端点 |
| 7 | **HTTP/3 支持** | 路由级别开关 |
| 8 | **上游重试次数 UI 化** | 目前 retries 字段存在但界面没有突出配置 |
| 9 | **WebSocket 支持** | 路由层面配置选项 |
| 10 | **访问日志查看** | 节点级别的访问日志查看和下载 |

### P2 - 体验改进

| # | 功能 | 说明 |
|---|---|---|
| 11 | **WAF 规则** | Web 应用防火墙规则配置 |
| 12 | **告警通知** | 节点离线/异常告警 |
| 13 | **灰度发布** | 先发一台验证后再全量 |
| 14 | **发布前自动对比** | 发布前自动对比差异，确认后再发 |
| 15 | **一键回滚** | 从发布记录直接回滚 |
| 16 | **插件依赖管理** | 插件之间的依赖关系管理 |
| 17 | **监控指标** | QPS、响应延迟、错误率等网关指标 |

## 四、现有系统可做的改进

### 4.1 路由匹配增强

当前路由匹配条件较少（域名/路径/请求头/方法）。Edge Admin 在"应用"层面支持更丰富的条件组合。建议在现有的 `vars` 高级条件基础上，增加常用条件的 UI 化配置面板。

### 4.2 发布流程优化

当前发布流程：选择节点 → 确认 → 执行。建议增加：

- **灰度发布**：先发一台，验证后再全量
- **版本对比**：发布前自动对比差异
- **发布回滚**：一键回滚到上一个版本

### 4.3 插件配置增强

- 当前插件编辑器有表单 + JSON 双模式，但表单覆盖率有限
- 建议增加**插件依赖管理**和**插件版本管理**

### 4.4 监控与可观测性

- 当前只有节点统计（CPU/内存/磁盘）
- 建议增加：**请求量 / QPS**、**响应延迟**、**错误率**等网关指标

## 五、建议优先级

```
第一梯队（P0-P1，预估 2-3 周）:
  □ 代理缓存 —— 路由缓存策略配置
  □ 响应压缩 —— Gzip / Brotli / Zstd
  □ CC 防护 —— CC 攻击日志与阈值配置

第二梯队（P1，预估 2-3 周）:
  □ gRPC 代理支持
  □ K8s 服务发现集成
  □ SSL/TLS 证书管理

第三梯队（P2，持续改进）:
  □ HTTP/3 支持
  □ 灰度发布
  □ 告警通知
  □ WAF 规则
  □ 监控指标
```
