# HTTPS/SSL 支持方案

> 基于现有 磐石 Admin 系统架构和 Edge 节点 API 能力，讨论 HTTPS 支持的方案设计。

---

## 背景

Edge 节点已支持 SSL 证书管理 API：
| 方法 | 路径 | 说明 |
|---|---|---|
| `PUT` | `/edge/admin/ssl/{id}` | 创建/替换证书 |
| `GET` | `/edge/admin/ssl` | 证书列表 |
| `GET` | `/edge/admin/ssl/{id}` | 查看证书 |
| `DELETE` | `/edge/admin/ssl/{id}` | 删除证书 |

证书数据结构：
```json
{
  "cert": "-----BEGIN CERTIFICATE-----...",
  "key": "-----BEGIN RSA PRIVATE KEY-----...",
  "snis": ["edge.local"],
  "type": "server",
  "ssl_protocols": ["TLSv1.2", "TLSv1.3"],
  "status": 1
}
```

---

### A2：集群范围 Tab 页

在**集群详情页**增加"SSL 证书"Tab，证书与集群绑定。

```
┌─ 集群: 生产集群 (ID:1) ─────────────────────────┐
│  [上游] [路由] [插件组] [全局规则] [SSL 证书]     │
│                                                   │
│  [+ 添加证书]                                     │
│                                                   │
│  ┌─────────────────────────────────────────────┐  │
│  │ 证书ID         │ SNI        │ 状态  │ 操作   │  │
│  │─────────────────────────────────────────────│  │
│  │ my-cert        │ edge.local │ ✅ 有效 │ 📝🗑 │  │
│  │ api-cert       │ api.com    │ ✅ 有效 │ 📝🗑 │  │
│  │ old-cert       │ test.local │ ⚠️过期 │ 📝🗑 │  │
│  └─────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────┘
```

**发布流程：** 证书推送到该集群的所有 Edge 节点，路由发布时引用集群证书。

**优点：** 证书与集群关联清晰，适合每个集群独立管理证书的场景。

**缺点：** 不支持跨集群共享证书；已存在的 PluginMetadata 在集群详情也有 Tab，但页面层级深。

### A3：路由内嵌（简化版）

在路由编辑弹窗中直接处理 SSL，不需要独立页面。

```
┌─ 编辑路由 ──────────────────────────────────────┐
│  [基础配置] [高级匹配] [插件] [SSL]              │
│                                                   │
│  ┌─ SSL ──────────────────────────────────────┐   │
│  │                                             │   │
│  │ 启用 HTTPS ☑                               │   │
│  │                                             │   │
│  │ SNI 域名 ─────── [edge.local]               │   │
│  │                                             │   │
│  │ 证书内容 (PEM)                               │   │
│  │ ┌────────────────────────────────────────┐   │   │
│  │ │ [上传文件] 或粘贴文本                   │   │   │
│  │ │ -----BEGIN CERTIFICATE-----             │   │   │
│  │ └────────────────────────────────────────┘   │   │
│  │                                             │   │
│  │ 私钥内容 (PEM)                               │   │
│  │ ┌────────────────────────────────────────┐   │   │
│  │ │ [上传文件] 或粘贴文本                   │   │   │
│  │ │ -----BEGIN PRIVATE KEY-----             │   │   │
│  │ └────────────────────────────────────────┘   │   │
│  │                                             │   │
│  │ 发布时 SSL 证书会一起推送到 Edge 节点        │   │
│  └─────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

**优点：** 无需额外页面，在路由编辑中一站式完成。
**缺点：** 每个路由都要重复上传证书，无法复用；如果多个路由用同一张证书，需要重复上传。

---

## 对比总结

| 维度 | A1 独立页面 | A2 集群 Tab | A3 路由内嵌 |
|---|---|---|---|
| 开发量 | ⭐⭐⭐ 中等 | ⭐⭐ 较小 | ⭐ 最小 |
| 证书复用 | ✅ 全局共享 | ⚠️ 集群内共享 | ❌ 无法复用 |
| 操作路径 | 证书页 + 路由页两步 | 集群详情内完成 | 路由编辑一步完成 |
| 适合场景 | 多路由共用证书 | 每个集群独立管理 | 少量简单路由 |
| 与现有模式一致性 | ✅ 类似 PluginMetadata | ✅ 类似集群内资源 | ⚠️ 增加路由表单复杂度 |

**推荐 A1**，与现有 PluginMetadata 管理页面模式一致，用户已熟悉这种操作流程。
