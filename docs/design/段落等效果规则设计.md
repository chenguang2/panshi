# 字段等效规则设计

> 讨论时间：2026-05-15
> 上下文：配置对比 Drawer 完成后，针对 DB → Edge 对比中字段等效性问题的设计方案

---

## 问题描述

数据库与 Edge 节点配置对比时，DB 只存储了必要的设置（不含默认值），Edge 返回的是完整展开的配置（含全部默认值）。直接字符串对比会产生大量误报差异。

### 等效问题类型

| 类型 | 举例 |
|---|---|
| **缺省值等效** | DB `load_balance=None`，Edge `"type": "roundrobin"` |
| **名称映射** | DB 叫 `load_balance`，Edge 叫 `type` |
| **JSON 子键缺省** | DB `timeout=NULL`，Edge `{"connect":15,"send":30,"read":30}` |
| **值格式差异** | DB `"GET,POST"`，Edge `["GET","POST"]` |
| **结构差异** | DB 用 `UpstreamTarget` 子表，Edge 用 `nodes: {"host:port": weight}` |
| **插件级默认** | 不同插件的 config JSON 各有不同默认值 |
| **Edge 内部元数据** | `create_time`、`update_time` 等在 Edge 上有，DB 没有 |

## 设计方案

### 架构

```
backend/app/
├── config/
│   └── equivalence_rules.yaml    → 规则文件（集中管理）
├── services/
│   └── config_diff.py            → EquivalenceRules 引擎
```

### 规则文件结构

```yaml
# backend/app/config/equivalence_rules.yaml

# ========================================
# 字段等效规则
# 规则说明：DB 侧缺失 / NULL 的字段，
# 如果等效于 Edge 侧某值，则视为一致
# ========================================

upstream:
  # DB 字段名 → Edge 字段名映射
  field_aliases:
    load_balance: type

  # 标量字段的默认值等效
  field_defaults:
    load_balance: roundrobin      # DB "weighted_roundrobin" → Edge "roundrobin"
    scheme: http
    pass_host: pass
    hash_on: vars
    retries: 5                    # 待确认：实际为动态值（节点数）

  # 从 Edge 侧存在、不应视为差异的标量字段
  ignore_edge_fields:
    - update_time
    - create_time
    - header
    - count

  # JSON 字段规则
  json_fields:
    timeout:
      fill_defaults:
        connect: 15
        send: 30
        read: 30
      ignore_edge_keys:
        - create_time
        - update_time
    keepalive_pool:
      fill_defaults:
        size: 32
        idle_timeout: 60
        requests: 10000
    checks:
      fill_defaults:
        active:
          http_path: /status
          timeout: 1
          healthy:
            http_failures: 0
            interval: 5
            successes: 1
          unhealthy:
            http_failures: 2
            interval: 5
            tcp_failures: 0
            timeouts: 0
          req_headers: []
          type: http


route:
  field_defaults:
    priority: 0
    status: 1

  # DB 逗号字符串 → Edge JSON 数组的字段
  list_fields:
    methods: split_comma
    hosts: split_comma

  ignore_edge_fields:
    - update_time
    - create_time
    - plugin_config_id


plugin_config:
  # 插件级规则：不同插件各有默认值
  json_field: plugins
  per_plugin_defaults:
    cors:
      allow_origins: "*"
      allow_methods: "*"
      allow_headers: "*"
      allow_credential: false
    limit-req:
      rate: 1
      burst: 0
      rejected_code: 503
    auth_key:
      hide_credentials: false
      header: apikey
      query: apikey
    proxy-rewrite: {}
    monitor:
      prefer_name: false
    traceid: {}
```

### 核心算法

#### JSON 字段填充-对比

```python
def _compare_json_field(db_val, edge_val, field_rules):
    """对比 JSON 字段，填充缺省值后再比较"""
    # 1. 解析
    db_parsed = json.loads(db_val) if isinstance(db_val, str) else db_val or {}
    edge_parsed = edge_val or {}

    # 2. 移除 Edge 内部字段
    for key in field_rules.get("ignore_edge_keys", []):
        edge_parsed.pop(key, None)

    # 3. 用 fill_defaults 递归填充 DB 缺失键
    defaults = field_rules.get("fill_defaults", {})
    db_filled = _deep_fill(db_parsed, defaults)

    # 4. 深比较
    if json.dumps(db_filled, sort_keys=True) != json.dumps(edge_parsed, sort_keys=True):
        return {"name": "...", "db": json.dumps(db_filled, indent=1),
                "edge": json.dumps(edge_parsed, indent=1)}
    return None


def _deep_fill(obj, defaults):
    """递归填充缺失的键，不覆盖已有值"""
    if not isinstance(obj, dict) or not isinstance(defaults, dict):
        return obj if obj is not None else defaults
    result = {}
    # 保留 obj 已有的键
    for k, v in obj.items():
        if k in defaults:
            result[k] = _deep_fill(v, defaults[k])
        else:
            result[k] = v
    # 补充 obj 缺失但 defaults 有的键
    for k, v in defaults.items():
        if k not in result:
            result[k] = deepcopy(v) if isinstance(v, dict) else v
    return result
```

#### 插件 JSON 二层拆分-对比

```python
def _compare_plugins(db_plugins, edge_plugins, plugin_defaults):
    """按插件名拆分后分别填充对比"""
    db = json.loads(db_plugins) if isinstance(db_plugins, str) else db_plugins or {}
    edge = edge_plugins or {}
    fields = []

    all_plugins = set(list(db.keys()) + list(edge.keys()))
    for name in all_plugins:
        db_conf = db.get(name, {})
        edge_conf = edge.get(name, {})
        defaults = plugin_defaults.get(name, {})
        db_filled = _deep_fill(db_conf, defaults)
        if json.dumps(db_filled, sort_keys=True) != json.dumps(edge_conf, sort_keys=True):
            fields.append({"name": f"plugins.{name}",
                           "db": json.dumps(db_filled, indent=1),
                           "edge": json.dumps(edge_conf, indent=1)})
    return fields
```

#### 规则引擎加载

```python
import yaml
from pathlib import Path
from copy import deepcopy

class EquivalenceRules:
    """加载和应用字段等效规则（单例）"""

    _instance = None
    _rules: dict = {}

    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
            cls._instance._load()
        return cls._instance

    def _load(self):
        path = Path(__file__).parent.parent / "config" / "equivalence_rules.yaml"
        if path.exists():
            with open(path) as f:
                self._rules = yaml.safe_load(f) or {}
        else:
            self._rules = {}

    def get_field_default(self, resource_type: str, field: str):
        return self._rules.get(resource_type, {}).get("field_defaults", {}).get(field)

    def get_json_field_rules(self, resource_type: str, field: str):
        """获取 JSON 字段的填充规则"""
        return self._rules.get(resource_type, {}).get("json_fields", {}).get(field, {})

    def should_ignore_edge_field(self, resource_type: str, field: str) -> bool:
        return field in self._rules.get(resource_type, {}).get("ignore_edge_fields", [])

    def get_field_alias(self, resource_type: str, field: str) -> str:
        return self._rules.get(resource_type, {}).get("field_aliases", {}).get(field, field)
```

### 关键原则

| 原则 | 说明 |
|---|---|
| **只补缺省，不覆盖** | `_deep_fill` 只添加 DB 缺失的键，不改 DB 已有的值 |
| **递归支持嵌套** | `timeout`、`checks` 等多层 JSON 都能处理 |
| **忽略 Edge 内部元数据** | `ignore_edge_keys` 过滤 `create_time` 等非配置字段 |
| **插件按 name 拆分** | 避免把不同插件的默认值搞混 |
| **先有框架，逐步收集** | 规则需在实际使用中逐步发现和完善 |

## 现有代码中发现的问题

这些也应一并修复：

| # | 问题 | 位置 | 说明 |
|---|---|---|---|
| 1 | `load_balance` 永远误报差异 | `clusters.py:1389` | DB `"weighted_roundrobin"` vs Edge `"roundrobin"`，`_UPSTREAM_DEFAULTS` 里写的是 DB 值，但 Edge 返回映射后的值 |
| 2 | `methods` 格式不同 | `clusters.py:1429` | DB 逗号字符串 `"GET,POST"` vs Edge 数组 `["GET","POST"]`，字符串比较永远失败 |
| 3 | `weight` 默认值不同 | `models/cluster.py:53` vs `edge_client.py:280` | DB `100` vs Edge `1` |
| 4 | `priority=0` 不发送到 Edge | `edge_client.py:349` | `if priority:` → 0 是 falsy，跳过不发送，导致路由优先级丢失 |
| 5 | JSON 解析 bug | `clusters.py:1382` | `startswith("{" or "[")` → 语法问题，`"{" or "["` 始终只判断 `{`，不会匹配 JSON 数组 |
| 6 | `hash_on` 缺省值未处理 | `clusters.py:1395` | DB `None` vs Edge `"vars"`，`_UPSTREAM_DEFAULTS` 中没有此键 |

## 收集路径

规则的填充需要在实际使用中逐步发现。建议流程：

```
部署框架 → 对比 → 发现误报 → 分析 Edge 返回值 → 写入规则 → 验证消除误报
```

初始阶段放已知规则（upstream 的 load_balance/scheme/pass_host + timeout），随使用逐步补充。
